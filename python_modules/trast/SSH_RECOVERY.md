# 🚨 ЭКСТРЕННОЕ ВОССТАНОВЛЕНИЕ SSH ПОСЛЕ WARP

## Проблема
После установки WARP SSH соединение заблокировано.

## Причина
WARP был установлен в режиме "warp" вместо "proxy", что перенаправляет весь трафик через Cloudflare.

## Решение

### 1. Быстрое исправление
```bash
sudo ./emergency_fix.sh
```

### 2. Ручное исправление
```bash
# Отключить WARP
warp-cli disconnect

# Установить безопасный режим
warp-cli mode proxy

# Проверить статус
warp-cli status
```

### 3. Если SSH все еще не работает
- Перезагрузите сервер через панель управления хостинга
- Используйте консоль хостинга для доступа
- Обратитесь в поддержку хостинга

## Предотвращение проблемы

### ✅ Безопасные команды
```bash
warp-cli set-mode proxy    # Только прокси (безопасно)
warp-cli connect           # Подключение
```

### ❌ Опасные команды
```bash
warp-cli set-mode warp     # Весь трафик (заблокирует SSH!)
```

## Режимы WARP

- **proxy** - Только прокси, SSH работает нормально
- **warp** - Весь трафик через Cloudflare, блокирует SSH

## Проверка режима
```bash
warp-cli settings
```

Должно показать: `Mode: proxy`
